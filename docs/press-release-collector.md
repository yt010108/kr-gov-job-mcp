# 기관 보도자료 collector 설계

조사일: 2026-07-07 KST

## 목적

기관 보도자료는 채용공고에는 드러나지 않는 최근 사업 방향, 협력 분야, 정책 키워드를
보강하는 근거가 될 수 있다. Job-ALIO 공고와 기관 분석을 연결할 때는 보도자료를 최종
판단이 아니라 `EvidenceSource` 후보로 보존한 뒤, 키워드와 날짜를 기준으로 선별하는 흐름이
안전하다.

## 샘플 기관 조사

| 기관 | 목록 URL | 상세 URL 패턴 | 관찰 |
| --- | --- | --- | --- |
| 한국인터넷진흥원 | `https://www.kisa.or.kr/402?page=1&searchDiv=10&searchWord=` | `/402/form?postSeq=...` | 목록 HTML 표에서 제목, 날짜, 상세 링크를 확인할 수 있다. |
| 창업진흥원 | `https://www.kised.or.kr` 하위 보도자료 메뉴 | `board.es?...act=view...` 계열 후보 | 홈페이지 메뉴에서 보도자료가 확인되며, board 계열 패턴 조사가 필요하다. |
| 국립공원공단 | 검색/뉴스·공지 하위 보도·설명 자료 | `boardDetail.do`, 파일 첨부 계열 후보 | 검색 결과에서 보도자료와 첨부파일 후보가 함께 노출된다. |

KISA 샘플 기준 최근 보도자료 목록은 공개 GET HTML만으로 수집 가능하다. 상세 본문도 같은
도메인의 HTML로 내려오며, 보안/개인정보/데이터/디지털 관련 키워드 매칭 후보를 만들 수
있다.

## 수집 범위 결정

1. 목록 페이지 HTML을 raw sample로 저장한다.
2. 목록에서 최근 상세 링크, 제목, 날짜를 metadata sample로 저장한다.
3. 상세 페이지 HTML을 raw sample로 저장한다.
4. 상세에서 제목, 등록일 후보, 본문 preview, 첨부파일 후보, 키워드 매칭 결과를 metadata
   sample로 저장한다.
5. 첨부파일은 후보 링크만 남기고 자동 다운로드하지 않는다.

## 필드 후보

| 필드 | 의미 | 사용처 |
| --- | --- | --- |
| `institution_name` | 기관명 | 기관 분석 대상 연결 |
| `title` | 보도자료 제목 | 근거 제목, 키워드 필터 |
| `published_date` | 등록일 | 최근성 필터 |
| `url` | 원문 상세 URL | 근거 출처 |
| `body_text_preview` | 본문 텍스트 일부 | 요약/키워드 검토 |
| `attachment_candidates` | 첨부 링크 후보 | 원문 보강 후보 |
| `matched_keywords` | 디지털/보안/데이터 등 키워드 | 직무 연결 후보 필터 |

## 키워드 필터

초기 키워드는 넓게 잡되, 보고서 생성 전 사람이 검토 가능한 후보로만 사용한다.

기본 후보:

- `디지털`
- `보안`
- `정보보호`
- `개인정보`
- `데이터`
- `전산`
- `사이버`
- `클라우드`
- `AI`
- `인공지능`
- `블록체인`
- `N2SF`
- `망 보안`

## EvidenceSource 후보

보도자료 상세 metadata에서 다음 구조를 만들 수 있다.

| 필드 | 값 |
| --- | --- |
| `source_type` | `press_release` |
| `institution_name` | 수집 query의 기관명 |
| `title` | 보도자료 제목 |
| `url` | 원문 상세 URL |
| `published_date` | 등록일 |
| `matched_keywords` | 키워드 매칭 결과 |
| `excerpt` | 본문 preview |

기관 사업 방향 분석에서는 이 후보를 ALIO 주요사업, 클린아이 사업보고서, 채용공고 직무기술서와
나란히 두고 교차 검증한다.

## 한계와 예외

| 한계 | 처리 |
| --- | --- |
| 기관마다 목록 HTML 구조가 다르다 | 목록 row와 상세 링크 힌트를 느슨하게 파싱하고, 반복 기관은 adapter를 둔다. |
| 검색/통합검색 결과가 섞일 수 있다 | `institution_name`, 목록 URL, 상세 URL 패턴을 metadata에 보존한다. |
| 메뉴/푸터 링크가 본문에 섞인다 | 첨부와 근거는 후보로만 저장하고 자동 확정하지 않는다. |
| 키워드 매칭은 맥락을 모른다 | 최종 보고서에서는 날짜, 제목, 본문 excerpt를 함께 검토한다. |
| 첨부파일 본문은 아직 읽지 않는다 | 파일 다운로드/문서 파싱 collector와 분리한다. |

## 구현 메모

`PressReleaseCollector`는 `list_url`과 `institution_name`을 받아 목록 HTML, 목록 metadata,
상세 HTML, 상세 metadata를 저장한다. KISA 샘플처럼 공개 HTML 표에서 제목/날짜/상세 링크가
노출되는 기관은 바로 최근 보도자료 후보 수집이 가능하다.
