# 공공기관 NCS 취업전략 MCP 제안서

## 1. 개요 및 목적

이 프로젝트는 공공기관 지원자가 채용공고, NCS 직무기술서, ALIO 경영공시, 클린아이 자료, 기관 최근 사업, 멘토링 피드백, 본인 경험을 한 번에 연결해 자기소개서와 면접 준비표를 만들 수 있게 하는 MCP입니다.

핵심 목적은 단순한 채용공고 추천이 아니라 공공기관 지원 전략을 근거 기반으로 생성하는 것입니다.

최종 사용자는 공고 URL이나 기관명을 입력하고 다음 결과를 받습니다.

- 공고 핵심 요약
- NCS 역량 매핑
- 기관 최근 사업 방향
- 기관의 부족한 점과 개선 포인트
- 사용자 경험과 공고 요구역량 매칭
- 자기소개서 문항별 소재 추천
- 예상 면접 질문과 꼬리질문
- 마지막 할 말 초안
- 참고 근거 링크

## 2. 배경 및 필요성

공공기관 취업 준비는 자료가 흩어져 있어 준비 난도가 높습니다.
지원자는 채용공고, 직무기술서, ALIO, 클린아이, 기관 홈페이지, 보도자료, 연구보고서, 국회 지적사항, 본인 경험 정리를 따로 확인해야 합니다.

문제는 자료 수집 자체보다, 이 자료들을 자기소개서와 면접 답변으로 연결하는 과정입니다.

공공기관은 KSA, 즉 Knowledge, Skill, Attitude 관점에서 역량을 평가하므로 공고와 본인 경험을 NCS 기준으로 구조화해야 합니다.

## 3. 목표와 성과

목표:

- 공공기관 전산직, 보안직, 데이터직 지원자를 위한 NCS 기반 준비 도구를 만듭니다.
- 채용공고와 기관 분석 자료를 연결해 지원 준비표를 자동 생성합니다.
- 사용자 경험과 멘토링 피드백을 반영해 자기소개서와 면접의 부족한 점을 진단합니다.
- 근거 링크가 있는 답변 전략을 제공해 단순 생성형 자기소개서와 차별화합니다.

핵심 결과물은 공공기관 지원 준비표입니다.

## 4. 방법 및 접근법

MCP는 다섯 단계로 동작합니다.

1. 공고 또는 기관명을 입력받습니다.
2. 채용공고와 직무기술서를 수집합니다.
3. ALIO, 클린아이, 기관 홈페이지, 연구보고서에서 기관 분석 자료를 수집합니다.
4. 공고 요구사항을 NCS/KSA 기준으로 구조화합니다.
5. 사용자 경험과 멘토링 피드백을 기준으로 강점과 부족한 점을 진단합니다.
6. 자기소개서 문항별 소재와 면접 답변 전략을 생성합니다.

멘토링 기반 분석 기준:

- 취업성공 4요소: 산업, 기업, 직무, 자기분석
- 기업 분석 3요소: 성장성, 안정성, 수익성
- 공공기관 분석 자료: ALIO, 클린아이, 잡알리오, 기관 홈페이지
- ALIO 활용 포인트: 채용정보, 주요사업, 국회 지적사항, 실시간 사업/외주 관련 정보, 연구보고서
- 면접 연결 방식: 주요사업은 지원동기, 국회 지적사항은 개선점 질문, 연구보고서는 마지막 할 말과 직무 관심 근거로 활용

## 5. 기대 효과 및 혜택

지원자 관점:

- 어떤 공고가 본인에게 맞는지 더 빨리 판단할 수 있습니다.
- 자기소개서에 쓸 경험을 NCS 역량 기준으로 정리할 수 있습니다.
- 기관 최근 사업 방향을 지원동기와 면접 답변에 연결할 수 있습니다.
- 기관의 부족한 점을 공격적으로 말하지 않고 개선 관점으로 표현할 수 있습니다.
- 멘토링 피드백을 반복적으로 적용할 수 있습니다.

프로젝트 관점:

- 단순 API 래퍼가 아니라 여러 출처와 개인 자료를 연결하는 실사용 MCP가 됩니다.
- 공공기관 채용이라는 한국 특화 문제를 해결합니다.
- 대화형 UI에서도 "이 공고 준비해줘"라는 명확한 사용 장면을 만들 수 있습니다.

## 6. 자원 및 예산

필요한 데이터 소스:

- 잡알리오 채용정보
- ALIO 경영공시
- 클린아이 지방공기업 정보
- 기관별 채용 페이지
- 기관 보도자료
- 기관 연구보고서
- 나라장터 입찰/외주 정보
- 사용자 이력서, 자소서, 프로젝트 경험, 멘토링 기록

필요한 기술 요소:

- 웹 문서 수집
- PDF/HTML 텍스트 추출
- 기관명 정규화
- NCS 역량 키워드 매핑
- 구조화 출력 스키마
- 사용자 경험 저장 및 검색
- 근거 링크 포함 답변 생성

## 7. 일정 관리

1단계. 조사 및 스키마 설계

- 대상 기관 3개 선정
- 공고/직무기술서 샘플 수집
- ALIO/클린아이에서 필요한 항목 확정
- NCS 역량 출력 포맷 정의

2단계. MVP 구현

- 공고 상세 수집 도구 구현
- 기관 분석 도구 구현
- NCS 매핑 도구 구현
- 사용자 경험 매칭 도구 구현
- 지원 준비표 생성 도구 구현

3단계. 검증

- KISA 기준 데모 생성
- 기존 멘토링 피드백과 비교
- 자기소개서 문항별 소재 추천 품질 확인
- 면접 질문과 꼬리질문 품질 확인

4단계. 확장

- 잡알리오/클린아이 검색 범위 확대
- 공모전/인턴 추천 기능 추가
- Notion 기반 지원 준비 히스토리 저장
- 면접 답변 채점 및 개선 기능 추가

## 8. 위험 및 대응책

- 자기소개서 대필 도구처럼 보일 수 있음: 작성보다 진단, 근거 연결, 소재 추천에 초점을 둡니다.
- 기관 분석이 부정확할 수 있음: ALIO, 클린아이, 기관 홈페이지 등 근거 링크를 함께 제공합니다.
- 마감일이나 채용 상태를 잘못 추정할 수 있음: 원문 공고에서 확인되지 않으면 확인 필요로 표시합니다.
- 국회 지적사항을 부정적으로 표현할 수 있음: 문제 지적이 아니라 개선 과제와 기여 방향으로 바꿔 표현합니다.
- 개인정보 처리 리스크가 있음: 사용자가 명시적으로 제공한 범위에서만 활용합니다.

- 사용자 경험과 요구역량 매칭 품질
- 부족한 점 진단의 구체성
